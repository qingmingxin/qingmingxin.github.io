Spring 学习——REST

发表于 2025-02-13 分类于 Spring

REST（资源表现层状态转换）是一种以资源为中心的架构风格，用 URI 唯一标识资源，通过 HTTP 动作（GET/POST/PUT/DELETE 等）实现资源的创建、查询、修改和删除。它以 JSON/XML 等格式表现资源状态，遵循无状态通信原则，强调 URI 设计的简洁性与语义化。在 Spring 中可通过注解快速构建符合 REST 风格的 API 端点，避免在 URI 中混入动词或版本号，让接口清晰易用。

Spring学习笔记——BeanFactory与ApplicationContext快速入门

发表于 2025-01-25 更新于 2025-02-13 分类于 Spring

BeanFactory 和 ApplicationContext 是 Spring 框架中管理 Bean 的两种主要接口。BeanFactory 是最基本的容器，采用延迟加载，只有在需要时才创建 Bean，适合简单应用和资源有限的环境。它提供基本的依赖注入，但不支持高级特性如事件传播和 AOP。
相比之下，ApplicationContext 是 BeanFactory 的扩展，提供更多功能。它在启动时预加载所有单例 Bean，并支持国际化、事件发布和 AOP 等特性，适合复杂的企业级应用。因此，通常推荐使用 ApplicationContext，因为它更灵活和功能强大。
